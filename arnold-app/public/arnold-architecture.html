Technical Debt — Resolved & Remaining

✓ Resolved — Phase 1: Storage Layer Unified

FIXED
Storage Bypasses Eliminated
All 4 bypass families fixed: profile, daily-logs, nutrition-log, supplements. 16 direct localStorage calls replaced with storage.get/set. All 20 keys now in KEYS map.
FIXED
Supplements Namespace + Sync
Supplements migrated from unprefixed keys to arnold:* namespace. DataSync key list corrected — supplements now sync between devices for the first time.

✓ Resolved — Phase 2a: Plan Completion Extracted

FIXED
checkTodayCompletion() in planner.js
3-store merge (garmin-activities + workouts + daily-logs) extracted from Arnold.jsx and MobileHome.jsx into a single shared function. ~40 duplicate lines removed.
FIXED
computeReadiness() in trainingIntelligence.js
General wellness readiness score (volume/pace/sleep/HRV/protein/bodyFat) extracted from MobileHome.jsx. Weighted formula now shared, available to Desktop too.

✓ Resolved — Phase A: Mobile Navigation

FIXED
Persistent Bottom Nav Bar
Bottom nav now stays visible when drilling into Training, Daily, and Body tabs. App-level nav renders when MobileHome is not active. "Recovery" renamed to "Daily".
FIXED
Swipe Gesture Navigation
useSwipeNav() hook added — horizontal swipe (>60px threshold) cycles through Home → Training → Daily → Body → Nutrition. Works both in MobileHome and drill-down tabs.

✓ Resolved — Phase B: Barcode Scanner

FIXED
Camera-Based Barcode Scanning
NutritionInput barcode mode now uses BarcodeDetector API with rear camera video feed. Detects EAN-13, UPC-A, Code 128 barcodes in real-time. Falls back to manual text input on unsupported browsers.

✓ Resolved — Phase C: Race Calendar Parity

FIXED
Race Countdown — Empty & Data States
Race countdown pill now renders in both empty-state and data-state MobileHome paths. Same-day races display "Today!" label. Race data computed consistently across all views.

✓ Resolved — Phase D: Activity → Nutrition Needs Engine

FIXED
activityNeeds.js Micro-Goals
New module computes replenishment targets (carbs pre/during/post, protein post, hydration) from activity stats (calories, duration, HR). Tracked against arnold:nutrition-log entries in ReplenishTracker component on LogDay tab.

Remaining — Phase 3 (Future)

DUPLICATE
Weekly Metrics (Phase 2c — parked)
Dashboard, LogDay, and TrainingTab each compute weekly miles/pace/volume independently. Logic varies slightly per context (some include strength, different time windows). Lower priority extraction.
SPLIT
Two Parallel Nutrition Systems
Old: arnold:cronometer (CSV import) → Dashboard nutrition panel
New: arnold:nutrition-log (manual/barcode/voice/photo) → NutritionInput display
dailyTotals() in nutrition.js falls back to Cronometer data, but they're displayed separately. Desktop LogDay shows both side by side. Fix: merge into a unified daily nutrition view.
TRIPLE
Three Activity Stores
arnold:garmin-activities — CSV bulk imports (historical)
arnold:workouts — Daily tab manual workout saves
arnold:daily-logs — FIT auto-persist (fitData nested object)
checkTodayCompletion() handles the merge, but no unified activity timeline exists yet. Fix: create getUnifiedTimeline(date) read-path consolidation.

Discovered During Phase 1 (Minor)

BYPASS
arnold:races — still 3 direct localStorage reads
TrainingTab lines 1012, 3038, 3115 in Arnold.jsx read localStorage directly despite races being in KEYS map.
MISSING
arnold:calendar-url, arnold:calendar-last-sync
Calendar URL and last sync timestamp stored directly in localStorage. Not in KEYS map. Low impact (RacesTab only).

⚠ Security Warning
All localStorage data is accessible to any JavaScript running on the same origin. AES-256-GCM encryption at rest is now active for HIGH and CRITICAL keys (hrv, sleep, weight, daily-logs, profile, backup slots). Session key stored in sessionStorage (cleared on tab close). Plaintext backups maintained for session key rotation.
